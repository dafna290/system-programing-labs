Вариант 6. Сервер принимает имя файла от клиента и,<br>
если файл существует, отправляет его содержимое.<br>

Используемые функции:<br>
socket() - создает сокет<br>
bind() - привязывает сокет к IP-адресу и порту<br>
listen() - переводит сокет в режим ожидания подключений<br>
accept() - принимает входящее соединение от клиента<br>
connect() -	установливает соединение с сервером<br>
send() - отправляет данные через установленное соединение<br>
recv() - получает данные из сокета<br>
close() - закрывает файловый дескриптор<br>
fork() - создает дочерний процесс<br>
stat() - проверка существования файла и получение информации о нем<br>
read() - чтение данныx<br>

Алгоритм работы программы:<br>
Инициализация функция main:<br>
Создает дочерний процесс через fork()<br>
Дочерний процесс запускает сервер<br>
Родительский процесс запускает клиент<br>
Родитель ожидает завершения дочернего процесса через wait()<br>

Серверная часть run_server():<br>
создает TCP-сокет<br>
устанавливает опцию SO_REUSEADDR для переиспользования порта<br>
привязывает сокет к порту<br>
переводит сокет в режим прослушивания<br>
блокируется до подключения клиента<br>
При подключении создается новый сокет client_fd для общения с этим клиентом<br>
Вызывается handle_client()<br>

handle_client():<br>
получает имя файла от клиента<br>
проверяет существование файла<br>
открывает файл (если существует)<br>
Цикл: read() из файла → send() клиенту<br>
отправляет маркер "EOF" для обозначения конца передачи<br>
закрывает файл и сокет клиента<br>

Клиентская часть run_client():<br>
создает клиентский сокет socket()<br>
преобразует IP-адрес "127.0.0.1" в бинарный формат<br>
устанавливает соединение с сервером<br>
читает имя файла с клавиатуры<br>
отправляет имя файла серверу<br>
