Задание 3-4: Диагностика сетевых соединений<br>

Использованные утилиты и их назначение:<br>

 `curl -I`  Получение HTTP заголовков<br>
 `curl -v`  Детальный HTTP запрос <br>
 `ping`  Проверка доступности <br>
 `ss` Просмотр сокетов<br>
 `netstat`  Сетевые соединения <br>
 `lsof`  Файлы и сокеты процесса <br>
 `nc`  Низкоуровневый TCP<br>

 Наблюдения:<br>
1. HTTP-заголовки (curl -I):<br>
- **Server**: SimpleHTTP/0.6 Python/3.12.3 - простой Python сервер<br>
- **Content-Type**: text/html - статические HTML файлы<br>
- **Content-Length**: 3661 байт - размер ответа (директория /home/dafna)<br>

2. Детали соединения (curl -v):<br>
- IPv6 connection refused → используется IPv4 (127.0.0.1)<br>
- HTTP 1.0 (сервер не поддерживает keep-alive)<br>
- Заголовки запроса: User-Agent, Accept<br>

3. Состояние порта (ss, netstat):<br>
- **Состояние**: LISTEN (ожидание подключений)<br>
- **Адрес**: 0.0.0.0:8080 (слушает на всех интерфейсах)<br>
- **PID/Program**: 419/python3<br>
- **TIME_WAIT** соединения после curl запросов<br>

4. Информация о процессе (lsof):<br>
- **COMMAND**: python3<br>
- **PID**: 419<br>
- **FD**: 3u (файловый дескриптор 3 для сокета)<br>
- **TYPE**: IPv4<br>
- **NAME**: *:http-alt (8080)<br>

5. Доступность (ping):<br>
- 0% packet loss - стабильное соединение<br>
- Время: 0.022-0.083ms - очень низкая задержка (localhost)<br>

6. Низкоуровневый анализ (nc):<br>
- Можно отправить сырой HTTP запрос<br>
- Сервер отвечает полным HTTP ответом с телом<br>
- HTML содержит директорию /home/dafna<br>
